# WeChat Post Publisher 微信公众号发布引擎

`wechat-post-publisher` 是一个为开发者和技术内容创作者量身打造的工程化发布技能 (Skill)。它旨在彻底打通从「本地 Markdown 极客撰写」到「微信公众号草稿箱」的自动化发布链路。告别繁琐的第三方网页工具复制粘贴、防止各种排版错乱和图片手动一张张上传的窘境，让你的重心彻底回归内容的创作。

---

## 🎯 核心原理 (Core Principles)

本工具并不仅限于一个简单的“文本 HTML 转换器”，它在内部构建了一条囊括状态隔离、素材热挂载、微信 API 强绑定的严谨流水线。

### 1. 深度定制的数据编译器 (Data Transformer)
微信公众号后台基于一套极其封闭受限的 HTML `style` 内联安全规范。为了让标准的 Markdown 无损过渡，底层注入了独立的转译引擎（即项目内的 `super-md`），并在编译节点做了两大劫持：
* **超链接接管**：将所有通用外链（微信除了授权白名单文章外不支持直接外链跳转）自动拦截，并重构为文末统一排版的 **「底部引文 (Citations)」** 体系。
* **AST 样式扁平化**：基于配置好的预设主题变量（如 `grace` 优雅风, `simple` 极简风），解析器会将对应的基础 CSS 样式树扁平化地计算并附着到每一个 HTML 底层元素的内容结构标签上，杜绝样式类污染。

### 2. 全自动多源媒体资产管控 (Autonomous Asset Uploads)
对于内容创作者最头疼的图片分发问题，引擎会动态生成一个正则抽象语法树提取所有的 `<img src>` 占位：
* **分离式上传机制**：对被识别到的正文插图使用 `media/uploadimg` 流量通道打到云端，无缝将本地原始路径替换为微信专属 CDN 地址。
* **封面板块隔离**：对封面图（无论是来自 Frontmatter 配置还是备用回退抓取的图 1），调用永久素材接口 (`material/add_material`) 做上传落库，拿回并挂载高权重的 `media_id` 以满足草稿图文规范结构的核心要求。

### 3. 多租户降维沙盒配置隔离 (Multi-Account Isolation)
系统通过极其克制的优先级层次模型覆盖状态机：
* **决策树优先级**：CLI 终端参数 `>` 文章 Markdown YAML 头部 `>` 项目微隔离配置（`.super-publisher/`） `>` 跨项目系统配置 （`~/.super-publisher/`）。
* **账户映射器**：并在 `EXTEND.md` 之中原生支持 `accounts` 多租户字典块配置，仅仅通过一个简短的别名（Alias）即可实现切换不同的 AppID 以及鉴权策略上下文，从机制上杜绝对个人号、企业生产号“发错文、配串号”的乌龙。

## 💡 落地最佳实践 (Best Practices)

### 1. 固化完善的 Frontmatter 元数据
要把 AI 和工具当做协同机器来用，最稳妥的实践是把文章的属性前置。建议所有的新建发贴都在首部加上完整的元数据块：

```yaml
---
title: "你的超赞文章标题：探索 AI 底层架构"
author: "超级发布团队"
summary: "千万字长文劝退？来看看这个工具带来的颠覆革命..."
coverImage: "./assets/cover-01.png"
---
```
拥有标准的头部元数据后，你只需要简单的触发一次工作流脚本，所有的验证拦截器都直接放行，极大加强流水线的无人值守能力（避免中断询问你缺失信息）。

### 2. 利用好系统原生相对路径 (Relative Pathing)
请避免在正文中写死磁盘的绝对路徑！建议你在文章同级目录下新建一个类似 `/images` 分组来存放插图。
技能引擎以 Markdown 所在的相对目录为根路由进行自动解包寻址：
```markdown
正确范式：![架构蓝图](./images/infra-map.png)
```
这能保证不仅在工具执行时平滑运作，更意味着你在不同的电脑之间拉取这个版本库（Git）也依旧能够做到完全通用。

### 3. 放心地使用无修饰的原生 Markdown 外链
写技术分享一定要带上代码地址或引申阅读，无需刻意逃避。针对微信的封控，本引擎自带的 Citation Transformer 已经搞定了一切。
当你无所顾忌地写下 `[阅读参考](https://github.com/guanyang/... )` 时，在最终排版发布之后，文中会巧妙地被加上一个右上角角标序号，并在文章尽头生成完美的「参考资料引文说明」底栏区。

### 4. 复杂环境执行防撞墙前排检查
如果在全新的办公设备第一次使用，或者在企业极客流 CI/CD 中拉起容器调用，你应当先借用它自带的脚本扫描仪排雷：
```bash
npx bun skills/wechat-post-publisher/scripts/check-permissions.ts
```
用来提前筛查底层编译器 Bun，以及剪接版、API 安全证书的健壮度合规水平。

## 🤖 智能化交互与使用范例 (Agent Prompts)

作为被大模型（如 Claude 或其他大语言终端）原生集成的极客技能 (Skill)，你不再需要亲手执行冷冰冰的代码，**更推荐使用纯正的自然语言或智能指令触发它**。大模型会为你自动处理所有的逻辑匹配与报错诊断。

以下是最高效唤起本系统工作流的 4 种主流（人机对话 Prompt）示例模式：

### 场景 1：极客风格的一键指令推流 (Slash Command)
如果你明确知道路径且不想废话，直接组合快捷指令串，Agent 便会静默替你走完全套流水线。
> `/wechat-post-publisher 文章 --markdown ./docs/article.md --theme modern`

### 场景 2：模糊参数推断式口语下发
抛弃对参数名刻板的记忆，让 AI 自己去理解意群要求并自动匹配属性集。
> *"帮我把项目根目录下的 `release-note.md` 发布到公众号草稿箱里去。由于这篇是关于架构重构的，排版我想要现代冷峻一点的感觉，主题帮我换成 `simple`，颜色指定黑白调。发布完把文章底稿链接告诉我！"*

### 场景 3：多租户身份的无缝脱靶式投递
依托于配置好的不同账户物理上下文（要求底层 `EXTEND.md` 存在 alias 别名编织）。
> *"这篇年终总结非常重要，我们要拿官方宣发号去发。去调那篇刚写完的文章发出去，同时千万记得在后台切走环境变量，用 `--account super-main` 的独立身份传过去哦！"*

### 场景 4：“创作->作图->排版上架”的超级闭环流
这是将业务 API 改造成 AI Skill 后真正的价值释放点！你完全可以让大模型成为独揽图文产研全栈的主编。
> *"昨天从 GitHub 拉取的这个项目架构有点复杂，请先通读它然后用大白话给我生成一篇 800 字的解析长文，然后顺便作一幅代表数据矩阵和代码交织的酷炫 16:9 配图存入 `./assets/` 中。整理完毕后，不再需要过问我，直接通过 `wechat-post-publisher`，把这图挂进头图里作为封面，以 grace 优雅风，提交到我的微信号后台做一个新草稿！"*
